ASConditionalTagHistoricalEntry
===============================

**Inherits:** ``RefCounted``

Historical entry for CONDITIONAL tag changes.

.. rst-class:: classref-introduction-group

Description
-----------

ASConditionalTagHistoricalEntry records when CONDITIONAL tags (requirement

and permission tags) are added or removed from a component. Stores the tag

name, target, timestamp, tick, and whether it was an addition or removal.



CONDITIONAL tags typically represent permissions and immunities like:

- Can.Parried, Can.Dodge, Can.Counter

- Immune.Fire, Immune.Poison, Immune.Stun

- State.Grounded, State.Flying (conditional movement states)



Used by the historical query system to track permission changes over

time, enable rollback functionality, and support combat mechanics that

depend on recent permission or immunity changes.



Example queries: "Was the player immune to fire recently?" or

"Did the player lose the ability to parry in the last second?"
